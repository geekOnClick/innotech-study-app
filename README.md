# Трекер задач

Учебное приложение на React + TypeScript. Данные приходят через моковый API без реального бекенда: методы возвращают `Promise` с небольшой задержкой, поэтому интерфейс ведет себя как при обычных сетевых запросах.

## Запуск

```bash
npm install
npm run dev
```

Если на Linux снова появится ошибка с количеством watchers, используй polling:

```bash
npm run dev:poll
```

Проект теперь собирается через Webpack, а не через Vite.

## Что есть в приложении

- загрузка стартовых задач через моковый запрос;
- создание, редактирование, удаление и переключение готовности задачи;
- фильтрация по статусу и приоритету;
- поиск по названию, описанию и исполнителю;
- счетчики задач и состояния загрузки/сохранения.

Моковый API находится в `src/mockApi.ts`.

## Учебные ошибки

В проект намеренно добавлены простые ошибки для тренировки поиска проблем через ChatGPT. Они отмечены комментариями `Учебная ошибка` в коде:

- слишком долгий ответ мокового API в `src/mockApi.ts`;
- неправильный подсчет активных задач в `src/App.tsx`;
- плохо видимый текст описания задачи в `src/styles.css`.
- в local storage устанавливается некорректный ключ apiStatusss в `src/App.tsx`;

## Шаблон обращения к ChatGPT

Лучше сначала описать симптом, а потом показать только нужный фрагмент кода. Полный проект отправлять не обязательно.

### Если проблема в запросе или загрузке данных

```text
У меня React + TypeScript приложение "Трекер задач".
Проблема: данные загружаются слишком долго.
Вот фрагмент файла:
[вставить код из src/mockApi.ts]

Подскажи, где именно может быть ошибка, почему она влияет на скорость, и как исправить ее простыми словами.
```

### Если проблема в логике

```text
У меня React + TypeScript приложение "Трекер задач".
Проблема: счетчик задач показывает неправильное значение.
Вот фрагмент файла:
[вставить код из src/App.tsx]

Найди ошибку в логике, объясни ее без сложных терминов и предложи минимальное исправление.
```

### Если проблема в верстке или стилях

```text
У меня React + TypeScript приложение "Трекер задач".
Проблема: текст описания задачи плохо читается.
Вот фрагмент файла:
[вставить код из src/styles.css]

Подскажи, какая стилeвая ошибка это вызывает, и покажи, как исправить ее понятным способом.
```

### Короткий универсальный запрос

```text
Помоги найти ошибку в этом фрагменте кода.
Сначала объясни, в чем проблема простыми словами.
Потом предложи минимальное исправление.
Если информации мало, задай мне уточняющий вопрос.
```
